Ejemplos de Aplicación de Calidad de Software

A continuación, se presentan casos prácticos que ilustran cómo las normas de calidad se aplican en proyectos de software reales para mejorar sus resultados.

Ejemplo 1: Programación sin aplicar normas ISO

Código (sin normas ni buenas prácticas):

# Calcula el promedio de notas (sin validaciones, sin estructura clara)
def promedio(notas):
  suma = 0
  for n in notas:
    suma += n
  return suma / len(notas)

notas = [5, 4, 2, 3, 1]
print(promedio(notas))

Problemas del código:

  • No tiene documentación ni comentarios técnicos claros.
  • No valida errores (por ejemplo, si la lista está vacía).
  • No sigue una estructura estandarizada de desarrollo.
  • No tiene nombres descriptivos ni formato de código uniforme.
  • No hay pruebas, control de versiones ni métricas de calidad.

Consecuencia:

El software puede funcionar inicialmente, pero será difícil de mantener, poco confiable y no cumplirá estándares de calidad exigidos por las organizaciones.

Ejemplo 2: Programación siguiendo normas ISO

Código mejorado (con buenas prácticas y estándares de calidad):

"""
Módulo: calculo_promedio.py
Descripción: Calcula el promedio de notas validando datos de entrada.
Normas aplicadas:
- ISO/IEC 25010: Calidad del producto (fiabilidad, mantenibilidad, usabilidad).
- ISO/IEC 12207: Proceso de desarrollo y documentación.
"""
def calcular_promedio(lista_notas):
    if not lista_notas:
        raise ValueError("La lista de notas no puede estar vacía.")
    if not all(0 <= n <= 5 for n in lista_notas):
        raise ValueError("Las notas deben estar entre 0 y 5.")
    promedio = sum(lista_notas) / len(lista_notas)
    return round(promedio, 2)
# Prueba unitaria básica (parte del proceso de verificación ISO/IEC 12207)
if __name__ == "__main__":
    try:
        notas_estudiante = [5, 4.5, 3, 4, 5]
        print("Promedio del estudiante:", calcular_promedio(notas_estudiante))
    except ValueError as e:
        print("Error:", e)

Buenas prácticas aplicadas:

  • Cumple con la documentación técnica y estructuración de código.
  • Incluye validaciones de entrada, mejorando la fiabilidad.
  • Usa nombres descriptivos y consistentes.
  • Integra una prueba unitaria básica para verificar el funcionamiento.

Resultado:

El código es seguro, mantenible, confiable y comprensible. Cumple estándares de calidad y facilita su evaluación.
